1. Подготовка структуры папок
Прежде чем открыть Krita, создайте на компьютере чёткую структуру папок. Это основа профессионального рабочего процесса.

Рекомендуемая структура:

text
/Карточный_проект/
├── /templates/         # Файлы-шаблоны раскладки (.kra или .ora)
├── /elements/          # Библиотека элементов
│   ├── /suits/         # Иконки мастей (черви, пики, бубны, трефы)
│   ├── /art/           # Изображения королей/дам/валетов/тузов
│   ├── /backgrounds/   # Фоны и рамки
│   └── /fonts/         # Шрифты (если используете TTF)
├── /cards/             # Готовые карты
│   ├── /hearts/        # Черви
│   ├── /spades/        # Пики
│   └── ...
└── /exports/           # Финальные PNG для печати или движка
2. Создание базового шаблона карты
Теперь создадим файл-основу для всех карт.

Шаг 2.1. Откройте Krita и создайте новый документ (File → New). Задайте размер вашей карты.
Для классических игральных карт популярен размер 600×800 пикселей (соотношение 3:4).
Укажите разрешение 300 ppi, если планируете печать .
Шаг 2.2. Сохраните файл как card_base.ora в папку /templates. Формат .ora (OpenRaster)
лучше сохраняет слои и совместим с другими программами .
3. Настройка направляющих и сетки
Это самый важный этап для точного позиционирования символов.
3.1. Включение панели сеток и направляющих
Шаг 3.1.1. Откройте панель Settings → Dockers → Grids and Guides .
У вас появится окно с двумя вкладками: Grid и Guides.
3.2. Настройка сетки (Grid)
На вкладке Grid настройте прямоугольную сетку :
Параметр	Значение	Пояснение
Show Grid	✓ (включено)	Отображать сетку
Snap to Grid	✓ (включено)	Включить привязку
Type	Rectangle	Прямоугольная сетка
X spacing	150 px	Ширина ячейки (600/4 = 150)
Y spacing	160 px	Высота ячейки (800/5 = 160)
Subdivision	1	Без дополнительных подразделений
Offset X/Y	0	Сетка от левого верхнего угла
Почему такие цифры? Я разделил ширину 600 на 4 колонки (150 px)
и высоту 800 на 5 рядов (160 px). Это даёт сетку 4×5, идеальную для
размещения символов на цифровых картах.
3.3. Создание направляющих (Guides)
Направляющие удобнее сетки, когда нужно отметить только ключевые линии .
Шаг 3.3.1. Включите линейки: View → Show Rulers .
Шаг 3.3.2. Включите отображение направляющих: View → Show Guides .
Шаг 3.3.3. Создайте направляющие, перетаскивая их из линеек:
Из левой линейки (вертикальные направляющие) — перетащите на координаты: 150, 300, 450 px
Из верхней линейки (горизонтальные направляющие) — перетащите на координаты: 160, 320, 480, 640 px
Важно: В текущей версии Krita нельзя задать точные координаты направляющих через диалог —
только перетаскиванием. Используйте сильное увеличение (zoom) для точного позиционирования .
Шаг 3.3.4. На вкладке Guides в панели Grids and Guides включите Snap to Guides и можете
заблокировать направляющие (Lock Guides), чтобы случайно не сдвинуть .
3.4. Настройка внешнего вида
На вкладках Grid и Guides измените стиль линий (Main Style), чтобы они не сливались с рисунком .
Я рекомендую для сетки сделать светло-серый пунктир, а для направляющих — ярко-синие сплошные линии.
3.5. Горячая клавиша для привязки
Запомните: Shift + S открывает всплывающее меню привязки, где можно быстро включать/выключать
привязку к сетке, направляющим, границам изображения и центру .
4. Создание шаблонов раскладки для разных рангов
Теперь, когда у нас есть базовая карта с сеткой, создадим отдельные шаблоны для разных типов карт.
4.1. Метод "слои-подложки"
Шаг 4.1.1. Возьмите файл card_base.ora и сохраните его как layout_10.ora в папку /templates.
Шаг 4.1.2. Создайте новый слой (назовите его "метки_10") и сделайте его полупрозрачным (20-30%
непрозрачности).
Шаг 4.1.3. На этом слое поставьте небольшие крестики или точки в тех местах, где должны стоять
символы масти для десятки:
Верхний ряд: позиции (1,2), (1,3), (1,4) по сетке 4×5
Нижний ряд: позиции (5,2), (5,3), (5,4)
Центральная группа: позиции (2,2), (2,3), (2,4), (3,2), (3,3), (3,4), (4,2), (4,3), (4,4)
Для десятки классическая раскладка — 10 символов (обычно 4+2+4 или симметрично).
Шаг 4.1.4. Сохраните файл.
4.2. Создание всех необходимых шаблонов
Повторите процесс для каждого ранга:
Файл шаблона	Для каких карт	Особенности раскладки
layout_2-6.ora	2, 3, 4, 5, 6	Симметрично сверху и снизу, центр пустой
layout_7-8.ora	7, 8	Как 2-6, но с 1-2 символами в центре
layout_9-10.ora	9, 10	Плотное заполнение, много символов
layout_face.ora	Валеты, дамы, короли	Только угловые индексы, центральная область для портрета
layout_ace.ora	Туз	Крупный символ в центре
5. Создание библиотеки элементов
Теперь создадим отдельные файлы для всех многократно используемых элементов.
5.1. Иконки мастей
Шаг 5.1.1. Создайте новый документ с прозрачным фоном. Размер иконки определите под свои карты
(например, 60×60 px для угловых символов, 120×120 для крупных).
Шаг 5.1.2. Нарисуйте символ черви. Сохраните как suit_heart.ora в папку /elements/suits/.
Шаг 5.1.3. Аналогично создайте:
suit_spade.ora (пики)
suit_diamond.ora (бубны)
suit_club.ora (трефы)
Важно: Рисуйте символы по центру холста и оставляйте вокруг немного прозрачного пространства —
это облегчит позиционирование при сборке.
5.2. Портреты для карт с картинками
Шаг 5.2.1. Импортируйте изображения королей, дам, валетов и тузов (можно нарисовать самим или
использовать готовые).
Шаг 5.2.2. Обрежьте и откадрируйте изображение под нужный размер (например, 400×500 px для
центральной области).
Шаг 5.2.3. Сохраните как king_hearts.ora, queen_spades.ora и т.д. в папку /elements/art/.
5.3. Фоны и рамки
Шаг 5.3.1. Создайте файл background_red.ora с фоном для червей и бубен.
Шаг 5.3.2. Создайте файл background_black.ora для пик и треф.
Шаг 5.3.3. Если хотите разные рамки для мастей, создайте отдельные файлы.
5.4. Шрифты для надписей
Krita использует системные шрифты. Установите нужные TTF-шрифты в операционную систему, и они
появятся в Krita.
Для угловых индексов (цифры и буквы) создайте отдельные файлы с текстом или используйте
текстовые слои непосредственно в картах.
6. Сборка финальной карты через File Layers
Вот где начинается магия автоматизации! Используем File Layers — слои, связанные с внешними файлами.
6.1. Создание карты десятки черви
Шаг 6.1.1. Создайте новый документ с размером карты (600×800 px). Сохраните как 10_hearts.ora
в папку /cards/hearts/.
Шаг 6.1.2. Импортируйте фон через File → Import/Export → Import File as Layer...
Выберите background_red.ora .
Шаг 6.1.3. Импортируйте шаблон раскладки как File Layer: выберите layout_10.ora.
Сделайте этот слой полупрозрачным (20-30%), чтобы видеть метки, но не мешали.
Шаг 6.1.4. Для каждой метки на шаблоне импортируйте иконку масти как File Layer:
Для верхних углов: импортируйте suit_heart.ora (уменьшите размер, если нужно)
Для центральных символов: импортируйте suit_heart.ora (обычный размер)
Повторите для всех 10 позиций
Шаг 6.1.5. Добавьте угловые индексы:
Создайте текстовый слой с цифрой "10" в левом верхнем углу
Создайте текстовый слой с цифрой "10" в правом нижнем углу (перевернутый)
Шаг 6.1.6. Когда все элементы расставлены, выключите видимость слоя с шаблоном раскладки.
6.2. Магия автоматического обновления
Теперь самое интересное. Если вы решите изменить дизайн иконки черви:
Откройте файл suit_heart.ora
Внесите изменения (например, сделайте символ более объемным)
Сохраните файл
Когда вы откроете 10_hearts.ora, Krita спросит: "Внешние файлы изменились. Перезагрузить?" — соглашайтесь
Все 10 иконок на карте обновятся автоматически!
То же самое с фонами, портретами и даже шаблонами раскладки.
6.3. Создание серии карт
Шаг 6.3.1. Создайте карту-образец для одного ранга (например, 10_hearts.ora).
Шаг 6.3.2. Скопируйте файл и переименуйте в 10_spades.ora.
Шаг 6.3.3. Откройте копию и замените File Layer с иконкой черви на иконку пики. Для этого:
Кликните правой кнопкой на слое с иконкой
Выберите Properties
В поле File укажите путь к suit_spade.ora
Шаг 6.3.4. Повторите для всех 4 мастей.
Шаг 6.3.5. Аналогично создайте образцы для других рангов (9, 8, 7 и т.д.).
7. Сохранение как официального шаблона Krita (опционально)
Если вы хотите, чтобы ваши шаблоны появлялись в диалоге File → New (как стандартные шаблоны
Krita для комиксов или анимации), сделайте следующее :
Шаг 7.1. Откройте ваш настроенный файл-шаблон (например, layout_10.ora со всеми направляющими).
Шаг 7.2. Выберите File → Create Template From Current Image .
Шаг 7.3. В диалоге:
Template Group: Создайте новую группу "Мои карты" или выберите существующую
Template Name: "10 (шаблон раскладки)"
Icon: Можно оставить автоматически сгенерированную иконку
Шаг 7.4. Нажмите Save.
Шаг 7.5. Повторите для всех ваших шаблонов раскладки.
Теперь при создании нового документа вы сможете выбрать группу "Мои карты" и сразу получить
готовую размеченную карту .
Альтернативный способ ручной установки
Если вы скачали готовые шаблоны из интернета :
Settings → Manage Resources... → Open Resource Folder
Откроется папка ресурсов Krita
Скопируйте папку с шаблонами в подпапку templates
Перезапустите Krita

Возможности плагина:

Выделите фигуру

Выберите пресет "Card" или настройте вручную
Плагин автоматически создаст направляющие и метки обреза — идеально для печати!

Способ 1: Горизонтальное отзеркаливание (классика)
Это стандартный подход, который мы уже обсуждали. Реализуется элементарно:

Шаг 1. Создайте File Layer с портретом (например, king_hearts.ora).
Шаг 2. Продублируйте слой: Layer → Duplicate Layer или Ctrl+J .
Шаг 3. Выберите нижний слой и примените отражение:
Image → Mirror Image Horizontally
Или используйте инструмент преобразования (клавиша T), кликните правой кнопкой и
выберите Flip Horizontally
Шаг 4. Переместите отраженную копию в нижнюю половину карты.

Плюсы: Просто, быстро, привычно для игроков.

🔄 Способ 2: Диагональное отзеркаливание (поворот на 180°)
Этот стиль создает интересный визуальный эффект — изображение в верхней половине
карты смотрит в ту же сторону, но находится «вверх ногами» относительно нижнего.
Технически это поворот на 180°, но с последующим перемещением в противоположный угол.

Шаг 1. Подготовьте File Layer с портретом (например, king_hearts.ora).
Расположите его в верхней части карты.
Шаг 2. Продублируйте слой: Layer → Duplicate Layer.
Шаг 3. Выделите дубликат и активируйте инструмент преобразования: клавиша T
или значок на панели инструментов .
Шаг 4. В опциях инструмента (панель Tool Options) найдите поле для ввода угла поворота. Введите 180°.
Шаг 5. Переместите повернутое изображение в нижнюю часть карты. Теперь оно будет смотреть в ту
же сторону, что и верхнее, но в перевернутом виде.
Альтернативный метод через Reference Images Tool:
Если вы работаете с референсами, в Krita 4.1+ есть удобный инструмент Reference Images Tool,
который позволяет свободно вращать и масштабировать изображения прямо на холсте :
Шаг 1. Tools → Reference Images → Add Reference Image .
Шаг 2. Загрузите портрет.
Шаг 3. Наведите курсор к углу изображения — появится значок вращения . Поверните на 180°.
Шаг 4. Расположите референс в нужном месте, а затем поверх него рисуйте или используйте как трафарет.
Важный нюанс: Reference Images Tool создает именно референсы, а не редактируемые слои.
Для финальной карты вам все равно понадобятся обычные слои или File Layers, но этот
инструмент отлично подходит для прицеливания и проверки композиции.
🔄 Способ 3: Вертикальное расположение без отзеркаливания
Это самый простой вариант — оба изображения одинаковы и ориентированы одинаково.
Шаг 1. Создайте File Layer с портретом.
Шаг 2. Продублируйте слой.
Шаг 3. Переместите копию в нижнюю часть карты.
Готово! Никаких трансформаций не требуется.
🎨 Комбинированный подход в вашей системе шаблонов
Чтобы интегрировать эти стили в созданную нами ранее систему шаблонов, сделайте следующее:
1. Создайте отдельные шаблоны для разных стилей
Файл шаблона	Для каких карт	Особенности
layout_face_horizontal.ora	Короли/дамы/валеты	Горизонтальное отзеркаливание
layout_face_diagonal.ora	Короли/дамы/валеты	Диагональное отражение (поворот 180°)
layout_face_vertical.ora	Короли/дамы/валеты	Вертикальное расположение без отражения
В каждом шаблоне на слое «метки» поставьте крестики там, где должны находиться:

Верхний портрет

Нижний портрет (с нужной ориентацией)

2. Для диагонального стиля — используйте два файла портретов
Поскольку поворот на 180° — это трансформация, которую нельзя применить к
File Layer «на лету» (связь с внешним файлом сохранится, но поворот —
это свойство слоя внутри документа), у вас есть два пути:

Путь А (простой): Создайте два File Layer с одним и тем же портретом,
но один из них поверните вручную через инструмент преобразования.
При обновлении исходного файла портрета оба слоя обновятся, но поворот останется.

Путь Б (более системный): Создайте два файла портретов:
king_hearts_normal.ora — обычная ориентация
king_hearts_rotated.ora — версия, повернутая на 180° и сохраненная
Тогда в карте вы используете:
Верхний портрет: File Layer → king_hearts_normal.ora
Нижний портрет: File Layer → king_hearts_rotated.ora
Недостаток пути Б: при изменении дизайна короля черви придется править два файла.

3. Для горизонтального отзеркаливания — используйте Flip Horizontally
Как в способе 1: продублируйте File Layer с портретом и примените Image →
Mirror Image Horizontally к копии.
