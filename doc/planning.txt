1.2. Проектирование формата данных

## Этап 2: **Python-сервер** (1 день)
**Цель:** Адаптировать существующий код для работы как сервер

2.1. Создание серверной обертки
   - Инициализация движка при старте
   - Обработка входящих запросов
   - Сериализация GameState в JSON

2.2. Эндпоинты API
   - `GET /state` - получить текущее состояние
   - `POST /move` - выполнить ход
   - `POST /draw` - взять карту
   - `POST /new` - новая игра
   - `POST /undo/redo` - отмена/повтор

2.3. Тестирование
   - Запуск сервера
   - Проверка через curl/Postman
